## Running locally

### Frontend (no build)

```bash
# from repo root
python3 -m http.server 8000
# open http://localhost:8000
```

Do **not** open `index.html` as a `file://` URL — the gallery uses HTTP 200/404 to probe for images.

VS Code Live Server also works.

### Backend

**Prerequisites:** Node.js 20+, MongoDB Atlas M0 (free), Gemini API key, Cloudinary free account.

```bash
cd backend
cp .env.example .env      # fill in all values — see table below
npm install
npm start                 # prestart copies templates, then starts on :3000
```

Dev mode (restarts on file changes):
```bash
npm run dev               # runs copyTemplates.js then node --watch server.js
```

Verify: `curl http://localhost:3000/healthz` → `{"status":"ok"}`

> **Critical gotcha — template sync**: `rate-card.html` (the Puppeteer template) is copied into `backend/templates/` only at server start via `scripts/copyTemplates.js`. Edits to `rate-card.html` are **invisible to Puppeteer** until the backend restarts or you run `node scripts/copyTemplates.js` manually.

## Data models

### User
`email` (unique, lowercase) · `passwordHash` · `name` · `bio` (max 80 chars) · `portfolioUrl` · `portraitUrl` · `photoLocked` (bool — set after portrait changed once via `/api/user/photo`) · `createdAt` / `updatedAt`

### Card (1-per-user)
`userId` (ref User, unique) · `rate` / `replaceability` (13–99) · `chessPiece` (pawn|knight|bishop|rook|queen|king) · `employmentStatus` (unemployed|intern|employed|retired) · `amCode` (4 letters + 4 digits) · `delta` `{value: float, direction: "up"|"down"}` · `educationOrg` / `workOrg` `{name, domain, logoUrl}` · `ctaUrl` · `linkedinUrl` · `portraitUrl` · `rawProfile` (Mixed — full LLM-parsed profile) · `imageUrl` (Cloudinary PNG URL) · `createdAt` / `updatedAt`

## Backend architecture

### Card generation pipeline

```
POST /api/card/generate
  → buildCardData(user, card)          cardData.js
  → renderCard(cardData)               puppeteer.js
      ├── inject window.CARD_DATA into rate-card.html (before </head>)
      ├── write to templates/_render/<uuid>.html
      ├── page.goto /_internal/template/_render/<uuid>.html
      ├── viewport: 1053×1470 px, deviceScaleFactor: 2  → 2106×2940 PNG
      └── screenshot #artboard element
  → uploadCardImage(buffer, userId)    cloudinary.js
      └── public_id: card_{userId}  (overwrites on re-generate)
  → save amCode + imageUrl + linkedinUrl to DB
```

### Score generation pipeline

```
POST /api/score/generate
  → scoreProfile(rawProfile)           scoring.js
      ├── complete(SYSTEM, prompt, { json: true })   llm.js → Gemini
      │     Returns: rate, replaceability, chessPiece, employmentStatus,
      │              delta, educationOrg, workOrg, bioRewrite
      ├── clamp rate + replaceability to [13, 99]
      └── getOrgLogo(eduName) + getOrgLogo(workName) in parallel
              └── Clearbit HEAD https://logo.clearbit.com/{slug}.com
                  → monogram SVG fallback if 404 / timeout
  → save all fields to Card document
```

### Auth flow (LinkedIn OAuth)

```
1. GET /api/auth/linkedin?intent=new
   → generates CSRF nonce (in-memory, 10-min TTL)
   → redirects to LinkedIn OIDC with state=base64(intent+nonce)

2. GET /api/auth/linkedin/callback  (LinkedIn redirects here)
   → validates state nonce
   → exchangeCode → access_token
   → getUserInfo → { email, name, picture }
   → upserts User in DB
   → uploadLinkedInPortrait to Cloudinary with e_upscale + quality:auto:best
   → stores full session payload in oauthSessions (2-min TTL, one-time)
   → redirects to FRONTEND_ORIGIN?oauth=<session_code>

3. GET /api/auth/linkedin/exchange?code=<session_code>  (frontend calls immediately)
   → consumes session code (deleted on first use)
   → returns { token, isNew, hasCard, name, photo, photoLocked, cardId, imageUrl, amCode }
```

### Puppeteer internals

- Singleton browser launched at server start (`initPuppeteer()` in `server.js`)
- Per-render: write temp HTML to `templates/_render/<uuid>.html`, navigate to it via `/_internal/template` static route, screenshot `#artboard`, then `page.close()` + delete temp file
- Page is closed immediately after screenshot — important for Render's 512 MB free tier
- Font loading awaited via `document.fonts.ready` before screenshot

### Cloudinary storage layout

| Folder | public_id | Content |
|---|---|---|
| `rate-cards/` | `card_{userId}` | Card PNG — overwritten on every re-generate |
| `portraits/` | `portrait_{userId}` | Portrait — overwritten on re-upload |

LinkedIn portraits are uploaded from URL then a transformation URL is returned (`e_upscale,q_auto:best`) — no local processing.

### Key utility functions

- `generateAmCode()` — 4 random uppercase letters + 4 random digits
- `replaceabilityToHourglass(n)` — maps score 13–99 to hourglass frame 1–10
- `statusToBlocks(status)` — maps employment status to 4 colored status chip configs
  - Index order: 0=retired (yellow), 1=intern (blue), 2=employed (green), 3=unemployed (red)

## Frontend architecture

### `index.html` — global config

```js
window.BACKEND_URL              = 'http://localhost:3000'
window.GALLERY_IMAGE_FOLDER     = './scatter-images/'
window.GALLERY_IMAGE_SCAN_LIMIT = 200
window.GALLERY_IMAGE_EXTENSIONS = ['jpg', 'jpeg', 'png', 'webp']
window.RATE_CARD_CAROUSEL_IMAGES = [...]   // 3-image hero carousel
window.RATE_CARD_CTA_URL        = ''       // optional hero CTA link
```

### `scatter-gallery.js` — 3D gallery + lead modal

**Rendering**: Custom perspective projection (`FOCAL = 920`), camera `(x, y, zoom)`, depth-sorted draw loop. All rendering is on a `<canvas>` that fills the viewport. The lead modal is the only DOM overlay.

**Lead modal state machine:**
```
entry
  ├── NEW USER → LinkedIn OAuth redirect (intent=new)
  │     └── on ?oauth=CODE return → confirm → upload-cv → loading → card
  └── RETURNING? → existing-login
        ├── LinkedIn (intent=existing) → loading → card
        └── email + amCode → loading → card
```

- `confirm` mode: collects **bio** and **portfolio/LinkedIn URL** only (name and photo come from LinkedIn, not editable here)
- `card` mode: shows "VIEW YOUR CARD →" which calls `openCardModal(amCode)`
- `openCardModal(amCode)`: full-screen iframe overlay loading `/card/:amCode?embed=1`; iframe dimensions computed from `window.innerHeight` so the card fits without scrolling; share button copies the card URL to clipboard
- JWT stored in `leadModal.token`; never persisted to localStorage
- Touch: `passive: false` on all touch listeners; pinch-to-zoom range 0.22×–3.2×

### `backend/public/card-view.html` — shareable card page

- Fetches `GET /api/card/view/:code` on load
- Renders the Cloudinary PNG with an invisible `<a>` overlay on the LinkedIn icon position (top: 80%, left: 5%, width: 90%, height: 6%)
- `?embed=1` hides the "CREATE YOUR OWN CARD" button — used when loaded in the modal iframe
- Card width: `min(clamp(240px, 92vw, 520px), calc((100vh - 160px) * 1053 / 1470))` — never requires scrolling

### `rate-card.html` — Puppeteer template

- 1053×1470 artboard; `transform: scale()` fits it to the container for browser preview
- `window.CARD_DATA` injected by Puppeteer before page load
- SVG `feConvolveMatrix` unsharp-mask applied to portrait `<img>` for sharpening
- Logo boxes show at most 3 org logos, left-to-right: educationOrg, workOrg
- Hourglass frames: `../assets/hourglass/hourglass-{1..10}.png`
- Chess pieces: `../assets/chess/{piece}.png`

## Admin panel (`/admin`)

- Single HTML file, no framework
- Token: 8-hour JWT stored in `localStorage.adminToken`; auto-logout on any 401 response
- **Users tab** loads on login/refresh; **Cards tab** loads lazily on first click
- Refresh (F5) only re-fetches the Users tab — much faster than loading both
- Cards table: amCode links directly to `/card/:amCode` for quick card preview; LinkedIn URL shown as link

## Deploy (Render)

```bash
cd backend
docker build -t raterace-backend .
docker run --env-file .env -p 3000:3000 raterace-backend
```

**Render settings**: Root Directory = `backend`, Start Command = `npm start`, add all env vars from the table above. Free 512 MB RAM instance is sufficient. Cold starts take ~15–20 s (Puppeteer browser launch).

`FRONTEND_ORIGIN` must be set to the deployed frontend URL for CORS and OAuth redirects to work correctly.
